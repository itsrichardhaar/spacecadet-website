## The problem

The ops team was drowning in a long tail of repeatable internal-tool tickets — account provisioning, role changes, integration toggles, subscription adjustments. Each took 5–15 minutes of mostly mechanical work, but every ticket needed enough context that no rules engine had ever worked.

The team had tried two off-the-shelf "agent" products. Both demoed beautifully and then quietly produced wrong actions in production with no observability into why.

## What we built

A multi-step agent that reads incoming tickets, pulls context from the company's existing internal systems, executes the action via owned tools, and writes a transcript back into the ticketing system for the human reviewer.

- Multi-step planning with explicit go / no-go gates on actions that touch billing
- Owned tool definitions — the agent calls the company's existing internal APIs, not opaque vendor adapters
- Eval suite of 180 historical tickets, classified by an ops lead, with golden expected actions
- Telemetry on every tool call, fed into Datadog with traces by ticket ID

## Approach

1. **Week 1 — Discovery.** Read the last six months of tickets. Identified the 23 ticket categories that covered 80% of volume.
2. **Weeks 2–3 — Eval suite.** Built the 180-case suite. Required ops lead sign-off on every golden expected action.
3. **Weeks 4–6 — Build.** Implemented the agent loop with LangGraph + Temporal for durable retries. Wrote the tool layer in pure Python — no vendor SDKs.
4. **Weeks 7–8 — Shadow run.** Agent ran on real tickets but only wrote suggested actions to a side queue for human approval. Eval kept rising as we tuned.
5. **Weeks 9–10 — Cutover + handoff.** Flipped to autonomous execution on the 14 lowest-risk categories. Kept the high-stakes 9 categories in human-approved mode by design.

## Outcome

71% of incoming tickets close automatically. The remaining 29% — the high-stakes ones — still go to a human, but with the agent's transcript and recommended action attached, so mean handle time on those dropped 47%. Eval suite has grown to 230 cases; the team owns it and runs it against every model upgrade.
